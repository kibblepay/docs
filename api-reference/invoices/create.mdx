---
title: "POST /api/invoices — Create a USDC Invoice"
description: "POST /api/invoices creates an invoice with line items, assigns a USDC deposit address, and optionally emails the PDF to your vendor on creation."
---

`POST /api/invoices` creates a new invoice, calculates the total from your line items, provisions a USDC deposit address, and optionally emails the PDF to your vendor immediately. Each invoice gets a unique hosted URL and a downloadable PDF endpoint.

The endpoint creates a merchant record the first time you use a given `merchant_email`, then auto-increments a per-merchant invoice counter to generate sequential invoice numbers like `INV-0001`.

<Note>
  By default, `send_now` is `true` — the vendor email with the PDF attachment is sent in the background immediately after the invoice is created. Set `send_now: false` to create the invoice in `"draft"` state without sending.
</Note>

## Request

**`POST https://pay.kibble.sh/api/invoices`**

### Body parameters

<ParamField body="merchant_email" type="string" required>
  Email of the merchant issuing the invoice. Used to create or look up a merchant account.
</ParamField>

<ParamField body="merchant_name" type="string" required>
  Display name of the merchant or business. Appears on the invoice. Minimum 1 character.
</ParamField>

<ParamField body="merchant_address" type="string" required>
  Physical or billing address of the merchant. Appears on the invoice. Minimum 1 character.
</ParamField>

<ParamField body="vendor_email" type="string" required>
  Email address of the vendor (recipient) who will receive the invoice.
</ParamField>

<ParamField body="vendor_name" type="string" required>
  Display name of the vendor. Appears on the invoice. Minimum 1 character.
</ParamField>

<ParamField body="vendor_address" type="string">
  Optional physical or billing address for the vendor. Appears on the invoice if provided.
</ParamField>

<ParamField body="line_items" type="object[]" required>
  Array of line items to include on the invoice. Must contain between 1 and 30 items. The total amount is derived by summing `quantity × unit_price` for all items.

  <Expandable title="line item properties">
    <ParamField body="description" type="string" required>
      A description of the product or service for this line item. Minimum 1 character.
    </ParamField>
    <ParamField body="quantity" type="number" required>
      Quantity of units. Must be a positive number.
    </ParamField>
    <ParamField body="unit_price" type="string" required>
      Price per unit in USDC as a decimal string (e.g. `"100.00"`). Must be a positive value.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="due_date" type="string" required>
  The payment due date in `YYYY-MM-DD` format.
</ParamField>

<ParamField body="issue_date" type="string">
  The invoice issue date in `YYYY-MM-DD` format. Defaults to today's date if omitted.
</ParamField>

<ParamField body="notes" type="string">
  Optional free-text notes to include at the bottom of the invoice.
</ParamField>

<ParamField body="wallet_type" type="string" required>
  How the USDC deposit address is sourced. One of `"byo"` (bring your own) or `"privy"` (Kibble-provisioned).
</ParamField>

<ParamField body="wallet_address" type="string">
  A valid EVM checksum address. Required when `wallet_type` is `"byo"`. Ignored when `wallet_type` is `"privy"`.
</ParamField>

<ParamField body="webhook_url" type="string">
  A valid HTTPS URL to receive a signed webhook notification when payment is confirmed. When provided, a `webhook_secret` is generated and returned in the response for HMAC verification.
</ParamField>

<ParamField body="send_now" type="boolean" default="true">
  Whether to immediately email the invoice PDF to `vendor_email`. Defaults to `true`. Set to `false` to create the invoice as a draft without sending.
</ParamField>

## Response

Returns `201 Created` with details about the newly created invoice.

<ResponseField name="invoice_id" type="string" required>
  UUID of the created invoice. Use this as the `{id}` parameter in all invoice sub-resource endpoints.
</ResponseField>

<ResponseField name="invoice_number" type="string" required>
  Sequential invoice number scoped to the merchant, e.g. `"INV-0001"`.
</ResponseField>

<ResponseField name="slug" type="string" required>
  An 8-character alphanumeric identifier for the invoice's hosted page.
</ResponseField>

<ResponseField name="invoice_url" type="string" required>
  URL of the hosted invoice page. Example: `https://pay.kibble.sh/i/xyz98765`.
</ResponseField>

<ResponseField name="pdf_url" type="string" required>
  URL to stream or download the invoice PDF. Example: `https://pay.kibble.sh/api/invoices/{id}/pdf`.
</ResponseField>

<ResponseField name="deposit_address" type="string" required>
  The EVM address (checksummed) that the vendor should send USDC to.
</ResponseField>

<ResponseField name="total_amount" type="string" required>
  Total invoice amount in USDC, derived from all line items, as a decimal string (e.g. `"500.00"`).
</ResponseField>

<ResponseField name="sent_at" type="string">
  ISO 8601 timestamp when the vendor email was sent. `null` if `send_now` was `false`.
</ResponseField>

<ResponseField name="webhook_secret" type="string">
  A 32-character alphanumeric secret for verifying signed webhooks. **Only present when `webhook_url` was provided.** Store this securely — it is not returned again.
</ResponseField>

## Errors

| Status | Description |
|--------|-------------|
| `400` | Validation failed — `error` contains a `fieldErrors` object. |
| `400` | `wallet_type` is `"byo"` but `wallet_address` is missing or not a valid EVM address. |
| `500` | Privy wallet provisioning failed. |
| `500` | Merchant record creation or invoice counter increment failed. |
| `500` | Unique slug could not be generated after 5 retries. |
| `500` | Database insert failed (payment link, invoice, or line items). |
