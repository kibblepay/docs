---
title: "GET /api/invoices/{id} — Get Invoice Details"
description: "GET /api/invoices/{id} returns the complete invoice record including all line items, merchant and vendor details, status, and a derived overdue boolean."
---

`GET /api/invoices/{id}` returns the complete invoice record, including all line items ordered by position. The response also includes a derived `overdue` boolean that is `true` when the due date has passed and the invoice status is still `"draft"` or `"sent"`.

<Note>
  The `webhook_secret` field is deliberately omitted from this response to avoid accidental exposure. It is only returned at invoice creation time.
</Note>

## Request

**`GET https://pay.kibble.sh/api/invoices/{id}`**

### Path parameters

<ParamField path="id" type="string" required>
  The UUID of the invoice, as returned in `invoice_id` from `POST /api/invoices`.
</ParamField>

## Response

Returns `200 OK` with the full invoice object.

<ResponseField name="id" type="string" required>
  UUID of the invoice.
</ResponseField>

<ResponseField name="invoice_number" type="string" required>
  Sequential invoice number, e.g. `"INV-0001"`.
</ResponseField>

<ResponseField name="slug" type="string" required>
  The 8-character slug used in the hosted invoice URL.
</ResponseField>

<ResponseField name="status" type="string" required>
  Current invoice status. One of `"draft"`, `"sent"`, `"paid"`, `"partial"`, or `"excess"`.
</ResponseField>

<ResponseField name="overdue" type="boolean" required>
  `true` if the due date is in the past and the invoice has not been paid (status is `"draft"` or `"sent"`). Computed at request time.
</ResponseField>

<ResponseField name="merchant_name_snapshot" type="string" required>
  The merchant name captured at the time the invoice was created.
</ResponseField>

<ResponseField name="merchant_address_snapshot" type="string" required>
  The merchant address captured at the time the invoice was created.
</ResponseField>

<ResponseField name="vendor_email" type="string" required>
  Email address of the vendor (invoice recipient).
</ResponseField>

<ResponseField name="vendor_name" type="string" required>
  Display name of the vendor.
</ResponseField>

<ResponseField name="vendor_address" type="string">
  Vendor address, if provided at creation. `null` otherwise.
</ResponseField>

<ResponseField name="issue_date" type="string" required>
  Invoice issue date in `YYYY-MM-DD` format.
</ResponseField>

<ResponseField name="due_date" type="string" required>
  Payment due date in `YYYY-MM-DD` format.
</ResponseField>

<ResponseField name="total_amount" type="string" required>
  Total USDC amount as a decimal string, e.g. `"500.00"`.
</ResponseField>

<ResponseField name="notes" type="string">
  Free-text notes included on the invoice. `null` if not set.
</ResponseField>

<ResponseField name="sent_at" type="string">
  ISO 8601 timestamp of when the vendor email was last sent. `null` if never sent.
</ResponseField>

<ResponseField name="webhook_url" type="string">
  The webhook URL registered at invoice creation, if any. `null` otherwise.
</ResponseField>

<ResponseField name="line_items" type="object[]" required>
  Ordered array of line items on the invoice.

  <Expandable title="line item properties">
    <ResponseField name="description" type="string" required>
      Description of the product or service.
    </ResponseField>
    <ResponseField name="quantity" type="number" required>
      Number of units.
    </ResponseField>
    <ResponseField name="unit_price" type="string" required>
      Price per unit in USDC as a decimal string.
    </ResponseField>
    <ResponseField name="line_total" type="string" required>
      Computed total for this line (`quantity × unit_price`), as a decimal string.
    </ResponseField>
    <ResponseField name="position" type="number" required>
      Zero-based display order of the line item.
    </ResponseField>
  </Expandable>
</ResponseField>

## Errors

| Status | Description |
|--------|-------------|
| `404` | No invoice found for the given ID. |

## Examples

<CodeGroup>

```bash cURL
curl --request GET \
  --url https://pay.kibble.sh/api/invoices/3f8a2c1e-4b9d-4e7f-a012-9c3d5e6f7a8b
```

```javascript JavaScript
const id = '3f8a2c1e-4b9d-4e7f-a012-9c3d5e6f7a8b';
const response = await fetch(`https://pay.kibble.sh/api/invoices/${id}`);
const invoice = await response.json();

if (invoice.overdue) {
  console.warn('Invoice is past due');
}
```

</CodeGroup>
