---
title: "POST /api/invoices/{id}/send — Re-send Invoice Email"
description: "Re-send the invoice PDF email to the vendor. Non-idempotent — each call sends a new email. Returns sent_at timestamp or an error field on delivery failure."
---

`POST /api/invoices/{id}/send` re-sends the vendor email with the invoice PDF attached. This is useful when the original email was not received, or when you want to send a follow-up reminder for an unpaid invoice.

<Warning>
  This endpoint is not idempotent — calling it multiple times sends a new email each time. The `sent_at` timestamp on the invoice is updated on every successful send.
</Warning>

## Request

**`POST https://pay.kibble.sh/api/invoices/{id}/send`**

This endpoint takes no request body.

### Path parameters

<ParamField path="id" type="string" required>
  The UUID of the invoice to re-send.
</ParamField>

## Response

Returns `200 OK` with the timestamp of the email send.

<ResponseField name="sent_at" type="string" required>
  ISO 8601 timestamp of when the email was sent in this request.
</ResponseField>

## Errors

| Status | Description |
|--------|-------------|
| `404` | No invoice found for the given ID. |
| `200` | Email delivery failed — the response body will contain `{ "error": "message" }` rather than `{ "sent_at": "..." }`. |

<Note>
  Email send failures return a `200` status with an `error` field in the body. Check for the presence of `sent_at` vs `error` in your response handling.
</Note>

## Examples

<CodeGroup>

```bash cURL
curl --request POST \
  --url https://pay.kibble.sh/api/invoices/3f8a2c1e-4b9d-4e7f-a012-9c3d5e6f7a8b/send
```

```javascript JavaScript
const id = '3f8a2c1e-4b9d-4e7f-a012-9c3d5e6f7a8b';
const response = await fetch(
  `https://pay.kibble.sh/api/invoices/${id}/send`,
  { method: 'POST' }
);
const data = await response.json();

if (data.error) {
  console.error('Email failed to send:', data.error);
} else {
  console.log('Invoice re-sent at:', data.sent_at);
}
```

</CodeGroup>

**Response — success:**

```json
{
  "sent_at": "2024-11-10T11:20:00.000Z"
}
```

**Response — email delivery failure:**

```json
{
  "error": "SMTP connection refused"
}
```
